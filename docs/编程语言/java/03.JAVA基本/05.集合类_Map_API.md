### 实现类

#### HashMap
```text
底层：数组 + 链表 + 红黑树
特点：无序、key 唯一、线程不安全、查询极快
允许 key=null、value=null（只能一个 null key）
适用：绝大多数单线程业务场景
```
#### LinkedHashMap
```text
继承 HashMap
特点：保留插入顺序，有序不排序
适用：需要存取顺序一致的场景、LRU 缓存

```
#### TreeMap
```text
底层：红黑树

特点：key 自动升序排序（自然排序 / 自定义比较器）

有序、不允许 null key

适用：需要按 key 排序的场景

```
#### Hashtable
```text
底层：哈希表

特点：线程安全、效率极低

不允许 key、value 为 null

已被 HashMap + 并发工具替代，不要用

```
#### ConcurrentHashMap
```text
JUC 并发包，线程安全

分段锁 / CAS+synchronized，并发性能远好于 Hashtable

适用：多线程高并发存键值对


```
